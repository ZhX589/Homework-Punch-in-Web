# 作业打卡网站开发计划

## 项目现状分析
- 现有代码：基于Flask的后端框架，使用SQLite数据库，已实现基本登录功能和数据库结构
- 缺失功能：学生登录完整实现、用户管理、CSV导入、作业管理、文件上传、小组管理、统计功能等

## 开发计划

### 1. 后端功能完善
- **完成学生登录功能**：实现姓名+班级+身份证后8位的登录验证
- **用户管理功能**：
  - 管理员添加教师和学生用户
  - 教师添加学生用户（手动和CSV导入）
  - CSV文件解析和批量导入学生
- **作业管理功能**：
  - 教师布置作业（标题、内容、截止日期、文件上传）
  - 学生提交作业（内容、文件上传）
  - 教师和组长评分功能
- **小组管理功能**：
  - 设置小组组长
  - 组长权限控制
- **统计功能**：
  - 作业完成情况统计（完成率、评分分布）
  - 按小组区分的统计数据
- **文件上传功能**：
  - 支持图片上传
  - 客户端压缩图片
  - 文件存储和管理

### 2. 前端开发
- **登录页面**：统一登录界面，支持管理员、教师、学生登录
- **管理员后台**：
  - 用户管理（添加、编辑、删除）
  - 班级管理
- **教师后台**：
  - 作业管理（布置、查看、评分）
  - 学生管理（手动添加、CSV导入）
  - 小组管理
  - 统计分析
- **学生后台**：
  - 作业查看和提交
  - 个人提交历史
- **组长后台**：
  - 同组同学作业查看和评分

### 3. 数据库优化
- 确保数据一致性
- 优化查询性能
- 添加必要的索引

### 4. 安全措施
- 密码加密存储
- 文件上传安全验证
- 权限控制加强

### 5. 测试和优化
- 功能测试
- 性能测试
- 用户体验优化

## 技术栈
- **后端**：Flask + SQLite
- **前端**：HTML5 + CSS3 + JavaScript + Bootstrap
- **文件存储**：本地文件系统
- **图片处理**：客户端压缩

## 开发重点
1. **CSV文件导入功能**：支持批量添加学生用户
2. **登录凭证设计**：姓名+班级+身份证后8位的组合验证
3. **小组管理和权限控制**：实现组长的特殊权限
4. **文件上传和管理**：支持图片上传和客户端压缩
5. **统计分析功能**：提供详细的作业完成情况统计

## 预期成果
- 完整的作业打卡网站，满足所有需求
- 良好的用户体验和性能
- 安全可靠的系统架构
- 易于维护和扩展的代码结构
